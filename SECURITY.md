# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 0.1.x   | Yes                |

## Reporting a Vulnerability

If you discover a security vulnerability in this provider, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please use one of the following methods:

1. **GitHub Security Advisories** (preferred): Go to the [Security tab](https://github.com/shoehorn-dev/terraform-provider-shoehorn/security/advisories) and click "Report a vulnerability"
2. **Email**: Send details to security@shoehorn.dev

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 5 business days
- **Fix timeline**: Depends on severity, typically within 30 days for critical issues

### Scope

This policy covers the Terraform provider code only. For vulnerabilities in the Shoehorn platform itself, please contact security@shoehorn.dev directly.
